# API endpoint reference

The application now exposes a lightweight REST API layer for dashboard, inventory, and feature navigation flows.

## Authentication
All listed endpoints require an authenticated user session.

## Endpoints

### GET /api/dashboard-summary/
Returns a lightweight summary payload used by the dashboard UI.

Fields:
- tape_count: total number of tape records
- shipment_count: total number of shipment records
- audit_log_count: total number of audit log entries
- pending_shipments: number of shipments still pending

### GET /api/tapes/
Returns the most recent tape inventory records for dashboard and integration use.

Fields per item:
- id: tape UUID
- volser: tape volume serial
- barcode: tape barcode
- status: current tape status
- current_location: current tape location
- retention_end_date: retention end date in ISO format
- tape_type: tape media type

### POST /api/tapes/
Creates a new tape record from a JSON payload.

Request body example:
```json
{
  "volser": "TAPE-100",
  "barcode": "BAR-100",
  "tape_type": "LTO-8",
  "status": "Active",
  "current_location": "Vault A",
  "retention_end_date": "2035-01-01"
}
```

Response:
- 201 Created with the created tape payload
- 400 Bad Request when validation fails

### GET /api/shipments/
Returns the latest shipment records for workflow monitoring and external integrations.

Fields per item:
- id: shipment UUID
- shipment_id: shipment identifier
- status: current shipment workflow status
- shipment_type: shipment category
- source_location: originating location
- destination_location: destination location
- priority_level: urgency level
- created_at: creation timestamp in ISO format

### GET /api/audit-logs/
Returns the latest audit trail entries for compliance and monitoring use cases.

Fields per item:
- id: audit log UUID
- name: audit log source name
- action: audit action description
- severity: log severity level
- timestamp: event timestamp in ISO format

### GET /api/features/<feature_key>/
Resolves a dashboard feature key to its standalone feature-module navigation payload.

Example:
- /api/features/operations_dashboard/

Response fields:
- feature_key: requested feature key
- target_url: feature-module URL that should be opened for the selected feature
- fragment_url: URL used to load the feature content as an embedded fragment
- load_mode: always page for standalone feature navigation
- scope: feature scope such as backup or operations

Behavior:
- Returns 401 Unauthorized when the user is not authenticated
- Returns 403 Forbidden when the user lacks access to the requested feature
- Returns 404 Not Found when the feature key does not exist

Example response:
```json
{
  "feature_key": "shipment_approvals",
  "target_url": "/features/shipment_approvals/",
  "fragment_url": "/features/shipment_approvals/?partial=1",
  "load_mode": "page",
  "scope": "operations"
}
```

### GET /api/investigation/<exception_id>/
Returns a read-only investigation payload for an open shipment exception.

Behavior:
- Accepts anonymous GET requests during development
- Returns 404 Not Found when the exception identifier does not exist
- Returns 400 Bad Request when the exception is not in an Open state
- Writes an audit entry for each access

Response sections:
- exception: exception identifier, type, status, severity, description, reporter, and reported timestamp
- tape: related tape metadata when available
- shipment: shipment status, locations, branch, custody information, and relevant user references
- timeline: transport events, receipts, approvals, and exception-reporting activity
- chain_of_custody: custody handoff and receipt information
- logs/audit_logs: investigation-related log history
- user_interactions: reporter and assigned context
- notifications: access and notification metadata
- related_exceptions: other exceptions tied to the same shipment
- users: relevant users and branch context
- branch_details: requesting branch metadata
- courier_details: courier name and vehicle number when available

Example:
- /api/investigation/EXC-3A0000D3/

### Notes
The same endpoint behavior is also exposed under the /apis/ prefix for compatibility with existing integrations.
